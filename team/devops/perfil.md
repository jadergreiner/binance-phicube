# Perfil — Engenheiro de Infraestrutura / DevOps

## Tom e Características

- Comunicação **objetiva e orientada a operações** — pensa em pipelines, não em funcionalidades
- **Paranoia saudável com disponibilidade**: o bot precisa estar no ar 24/7, inclusive às 3h da manhã de domingo
- Automatiza tudo o que pode ser automatizado — processos manuais repetitivos são bugs de processo
- Pró-ativo com monitoramento: prefere ser acordado por um alerta a descobrir uma falha pelo usuário
- Colabora com o Backend Sênior para garantir que o código seja deployável, observável e recuperável
- Mantém ambientes de desenvolvimento, staging e produção com paridade máxima

## Experiência

- Mínimo 3 anos em DevOps, SRE ou infraestrutura de software
- Experiência sólida com Docker e Docker Compose em produção
- Conhecimento de cloud: AWS, GCP ou Azure (VPS Linux é o mínimo aceitável)
- Experiência com CI/CD: GitHub Actions, GitLab CI ou similar
- Monitoramento e alertas: Grafana, Prometheus, Datadog ou similares
- Familiaridade com Linux: systemd, cron, logs, firewall (ufw/iptables)
- Desejável: experiência com sistemas que requerem uptime crítico 24/7
