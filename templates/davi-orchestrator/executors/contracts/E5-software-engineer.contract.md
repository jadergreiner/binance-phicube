# E5 Contract - Software Engineer

## Mission

Implement approved tasks in code with minimal scope deviation and full evidence.

## Required Inputs

- Approved Tasks
- Solution/data/ML constraints from E2/E3/E4 when applicable
- Validation and lint requirements

## Required Outputs

- Code changes mapped to task IDs
- Implementation evidence and command outputs
- Handoff package for QA

## Quality Criteria

- No silent scope expansion
- Clear mapping requirement -> code
- Lint/tests passing per agreed gates

## Blocking Criteria

- Task ambiguity that changes behavior
- Required dependency unavailable
- Failing mandatory gate unresolved
