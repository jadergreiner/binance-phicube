# E4 Contract - Machine Learning Engineer

## Mission

Implement and validate ML-specific changes
(training/inference/policies) within approved scope.

## Required Inputs

- Approved SPEC/Tasks with ML impact
- Current model/runtime constraints
- Risk and governance requirements

## Required Outputs

- ML implementation notes
- Validation evidence for model/runtime behavior
- Risk-aware rollout guidance

## Quality Criteria

- Change is reproducible and scoped
- Runtime behavior is measurable
- Safety constraints are enforced

## Blocking Criteria

- Missing evaluation criteria
- Unclear rollback/fallback in high-risk change
- Data/feature contract mismatch
